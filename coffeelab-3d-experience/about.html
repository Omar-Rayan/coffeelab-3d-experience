About This Project

How I made it, design choices, accessibility, and references.

1. How I Made the 3D Models

I built all three models in Blender 4.x and exported them as glTF (.glb) files for the web. I named each mesh properly (like Plunger, Knob, CupBody, BagBody) so the JavaScript could target them for animations, colour swaps, and the brewing colour-LERP.

French Press

Parts: Carafe, TopRing, BottomRing, Handle, Base, Filter, Rod, Plunger, Knob, Liquid (plus 50 coffee Grounds + Crema layer)

I made the carafe from a lathe profile and gave it glass material with transmission and IOR 1.45 so it looks realistic. The steel rings, handle, and base have low roughness and high metalness. The handle started as a Bezier curve, then I bevelled and converted it to mesh.

The animation shows the plunger going down, the liquid turning coffee-brown via a JS colour LERP, and steam particles rising.

Takeaway Cup

4 parts: CupBody, Sleeve, DomeLid, Straw

I made the cup as a tapered cylinder with a Solidify modifier so the walls have real thickness. The cardboard sleeve is a separate ring with interpolated radius. The dome lid is half a UV sphere with a torus lip. The straw is tilted and has a boolean difference for the hollow center.

Coffee Bag

Parts: BagBody + 256 individually-modelled beans inside + an Aroma particle group

The bag is a stand-up pouch with a soft bulge so it looks full. The opening animation is driven by a Blender shape key on the BagBody mesh, where vertices on the top seal morph apart to reveal the beans. The beans were modelled with bezier curve sweeps for a real coffee-bean silhouette and placed in a deterministic grid so they actually sit inside the pouch.

I exported everything as glTF 2.0 binary so meshes, materials and animations all travel in one file. I kept the poly counts low (few thousand tris per model) so it loads fast even on phones.

2. Design Choices

Why Three.js instead of X3DOM

  • Way more control over the scene, lights and materials
  • Bigger community and better docs
  • Modern features I could actually show off like PBR, bloom, custom shaders, post-processing

Web-friendly modelling

  • Kept each model to a few thousand triangles so it loads quick
  • Named everything consistently so colour swap and animations were easier to code
  • Used modifiers (Solidify, Boolean, Bevel) for clean topology

MVC architecture

  • Model (js/model.js) fetches products.json, holds product data, camera presets, colours, finishes
  • View (js/view.js) handles the Three.js scene, lighting, post-processing, GLTF loading, and animations
  • Controller (js/controller.js) connects DOM events to View methods and renders the UI from Model data
  • Each part is separate, so I could change one without breaking the others

Custom GLSL shaders

  • Cinematic pass (js/cinematic.js) is a fragment shader with chromatic aberration, film grain, and vignette. Added to the EffectComposer after Bloom
  • Steam particles (js/steam.js) is a vertex + fragment shader driving around 220 shader-driven particles (single-draw-call THREE.Points) with a sine-wobble lifecycle, additively blended for soft volumetric steam

UI design

  • CoffeeLab green (#00704A) as the main colour, with cream and gold accents
  • Sodo Sans font, clean and modern
  • All three control panels (Camera, Lighting, Colour) are open by default so nothing is hidden
  • Responsive Bootstrap 5 grid so it stacks properly on mobile

Storytelling animation

The Brew button doesn't just spin the model. The plunger goes down over about 2.5 seconds (baked Blender keyframes), the liquid inside turns coffee-brown via a JS colour LERP, steam particles fade in and out, and a brewing sound plays in sync (playback rate adjusted so the audio finishes exactly when the animation does). The Coffee Bag has its own opening animation driven by a shape key. I wanted the interactions to tell a little story about each product, not just rotate them.

Accessibility

I wanted the site to be usable for more than just mouse-and-keyboard desktop users:

  • Text contrast checked against WCAG AA (4.5:1 body, 3:1 headings)
  • Buttons use plain-text labels ("Brew Coffee", "Front", "Side") so meaning isn't icon-only
  • All controls are native <button>/<input> elements, so they work with keyboard and screen readers out of the box
  • Semantic HTML landmarks (<header>, <main>, <nav>, <footer>) and aria-labels on short controls (like colour swatches)
  • Bootstrap 5 grid collapses to single column under 992px, tested down to 375px

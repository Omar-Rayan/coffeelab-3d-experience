This project runs locally with Live Server in VS Code. Just open index.html with Live Server (or run python -m http.server). Three.js loads from CDN, all assets are relative paths, and JSON data is fetched at runtime. No build steps needed.

5. Deeper Understanding Statement

What I did beyond the lab tutorials

Custom GLSL shaders. I wrote two original shaders that go beyond what we covered in labs. js/cinematic.js is a fragment shader for chromatic aberration, film grain, and vignette. It's added to the EffectComposer chain after Bloom and before FXAA. js/steam.js is a vertex+fragment particle shader driving around 220 shader-driven steam particles (single-draw-call THREE.Points) with sine-wobble drift and additive blending.

MVC architecture. Instead of one big file like in the labs, I split everything into Model-View-Controller (js/model.js, js/view.js, js/controller.js). The Model fetches data from products.json using AJAX, so the data layer is cleanly separated from the rendering and UI code.

Storytelling animation. The Brew button doesn't just rotate the model. It runs a multi-step animation where the plunger descends over 2.5 seconds via baked Blender keyframes, the liquid mesh tints coffee-brown via a JS colour LERP, steam particles fade in and out, and a brewing sound plays in sync (with the audio playback rate adjusted to match the animation duration). This makes the interaction feel like part of the product's story rather than just a rotating object.

Bootstrap 5 + responsive design + post-processing chain show I went beyond basic lab work with proper frontend framework usage and advanced rendering.
